# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with this Obsidian vault.

## Overview

A personal knowledge base built on the Evergreen Notes methodology and the Think to Text Protocol — the principle that humans should capture their thinking in plain text, and collaborate with AI to transform it into deliverables.

## Vault Structure

- **Inbox/** — Fleeting notes, unsorted quick captures
- **Notes/** — Evergreen notes: thoughts digested in your own words. Default folder for new files
- **Daily/** — Daily journal. Filename format: `YYYY-MM-DD.md`
- **Projects/** — Project workspaces. Each project has a subfolder with the following structure:
  - `Product.md` — Latest spec (Single Source of Truth). Always keep current
  - `Project.md` — Overview, goals, tasks
  - `Archives/` — Stale data goes here. Old meeting notes, outdated specs, anything that would hurt the freshness of Product.md
- **Clippings/** — Articles saved via web clipper
- **Emails/** — Email archives
- **References/** — Reference cards for people, books, concepts
- **Generated/** — AI-generated content (distinct from your personal knowledge)
  - **Generated/Context/** — AI-maintained context documents
  - **Generated/Knowledge/** — AI-built knowledge artifacts
  - **Generated/Sessions/** — AI conversation session records
  - **Generated/Logs/** — Tool operation logs
- **Libraries/** — Vault infrastructure
  - **Libraries/Templates/** — Templater templates (folder templates enabled)
  - **Libraries/Bases/** — Bases plugin view definitions (`.base`)
  - **Libraries/Categories/** — Category index pages
  - **Libraries/Binaries/** — Attachments (images, files)

## Author / Editor Properties

All notes have an `author` frontmatter property to track who wrote them:
- `author: yourname` — Written by the vault owner
- `author: claude` — Generated by AI (mainly under `Generated/`)

When a different entity edits, use `editor`:
- `editor: claude` — Human's thoughts structured/edited by AI
- `editor: yourname` — AI-generated content reviewed by human

## Note Promotion Flow

Inbox (Fleeting) → Notes (Evergreen). A note promotes to Notes when it sheds source dependency and becomes thought expressed in your own words.

## Frontmatter by Folder

| Folder | Required frontmatter |
|---|---|
| Notes/ | `categories: ["[[Notes]]"]`, `tags`, `created`, `updated`, `published` (optional), `author`, `editor` |
| Projects/ | `categories: ["[[Projects]]"]`, `status: active`, `tags`, `created`, `updated`, `author`, `editor` |
| Daily/ | `categories: ["[[Daily]]"]`, `created`, `updated`, `author`, `editor` |
| References/ | `categories: ["[[References]]"]`, `tags`, `aliases`, `created`, `updated`, `author`, `rating` |
| Inbox/ | `categories: inbox`, `tags`, `created`, `updated`, `author` |

## Plugins

- **Templater** — Template engine with folder templates (auto-apply on file creation)
- **Tasks** — Dataview format. Custom statuses: `/` = In Progress, `-` = Cancelled
- **Linter** — Auto-format on save. Manages `created`/`updated` timestamps
- **Bases** — Core plugin. View definitions in `Libraries/Bases/`

## Editing Rules

- Preserve existing YAML frontmatter
- Use `[[wikilinks]]` for internal links
- Use `#tag` for inline tags
- Daily notes follow the template: frontmatter with `categories: ["[[Daily]]"]`, then `[[YYYY-MM-DD]] の記録`

## File Naming

- **English names** → PascalCase (`AiWorkflow.md`)
- **Japanese names** → as-is (`疎結合設計.md`)
- **Date-prefixed** → hyphen-separated date, then naming rule for title
